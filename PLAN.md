# 旋转地球网页项目计划（Plan Mode）

## 1. 目标
- 构建一个视觉精美、可交互、性能稳定的「旋转地球」单页网站。
- 支持桌面和移动端浏览。
- 项目通过 GitHub 管理，并可发布到 GitHub Pages。

## 2. 第一版范围（MVP）
- 3D 地球持续旋转（可暂停/继续）。
- 白天纹理 + 夜晚灯光纹理混合。
- 云层缓慢漂移。
- 背景星空与柔和光晕。
- 鼠标/触控拖拽旋转，滚轮缩放。
- 顶部简洁标题与说明。

## 3. 技术方案
- 核心渲染：`Three.js`
- 页面构建：原生 `HTML/CSS/JS`（先不引入框架，减轻复杂度）
- 动画控制：`requestAnimationFrame`
- 资源组织：
  - `src/main.js`：场景与动画主逻辑
  - `src/styles.css`：视觉样式
  - `assets/textures/*`：地球、云层、夜景、星空贴图

## 4. 视觉设计方向
- 主题关键词：深空、精致、高对比、低噪点。
- 色彩：深蓝黑背景 + 冷白高光 + 少量青色辉光。
- 细节：
  - 地球边缘 Fresnel 光效（增强球体体积感）
  - 背景分层星空（前后景深差）
  - 轻微镜头呼吸感（非常弱，避免眩晕）

## 5. 性能与质量
- 目标帧率：桌面 60fps，移动端尽量稳定在 30fps+。
- 贴图压缩与尺寸控制（优先 WebP / KTX2 视情况决定）。
- `devicePixelRatio` 上限控制（例如 1.5~2）避免高分屏过载。
- 提供降级策略（低性能设备可关闭部分后处理效果）。

## 6. 项目里程碑
1. 仓库初始化与远端同步（当前阶段）
2. 页面骨架 + Three.js 场景跑通
3. 地球材质、云层与光效细化
4. 交互与响应式适配
5. 性能优化与跨端测试
6. GitHub Pages 发布

## 7. GitHub Pages 发布计划
- 默认使用 `main` 分支。
- 使用 `docs/` 或 `gh-pages` 分支发布（二选一，推荐 `gh-pages`）。
- 每次合并到主分支后自动部署（后续可加 GitHub Actions）。

## 8. 待你确认的决策
- 仓库名是否固定为：`rotating-earth-web`
- 仓库可见性：`public` 或 `private`
- 是否使用 GitHub Actions 自动部署
